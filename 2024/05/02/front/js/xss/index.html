前言

主要围绕xss的利用方法和挖掘思路

分类

反射型

黑客往往需要诱使用户“点击”一个恶意链接，才能攻击成功

储存型

如写评论，存在数据库中，别人看评论时，调用数据库直接拼接，触发xss

DOM型

可以看成特殊的反射型，通过修改页面的DOM节点形成的XSS，称之为DOM Based XSS。

如果是将传入参数直接拼接到js文件中的化，有一个万能payload"-alert(1)-",有waf可以把（）换成``

JavaScript伪协议

  1. dom xss还可以用JavaScript伪协议，并且HTML在渲染属性的时候，会先解实体编码后才执行属性内部的JavaScript代码，所以并不会收到Html entity编码的影响，
    <a href="javascript:console.log('&#x27;&#x2b;&#x61;&#x6c;&#x65;&#x72;&#x74;&#x28;&#x31;&#x29;&#x2b;&#x27;')">testLink</a>
  2. 浏览器在渲染JavaScript伪协议地址的时候，会先进行URL解码，再执行JavaScript。
