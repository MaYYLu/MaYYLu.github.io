Java 字节码

Java也是一门编译型的语言，在运行之前需要先编译一遍，编译过后就会形成.class文件，然后运行在不同平台的JVM虚拟机中
所以字节码就是能在JVM虚拟机中加载成一个类的字节序列

classloader（类加载机制）

Java类初始化的时候会调用java.lang.ClassLoader加载类字节码,AppClassLoader（系统类加载器）是默认的类加载器

ClassLoader调用核心方法的顺序为：

  1. loadClass（从已经加载的类缓存、父加载器等位置去寻找类）
  2. findClass（若未找到，则根据指定路径的方式，在本地文件系统、jar包或远程http服务器上读取字节码）
  3. defineClass（处理前面传入的字节码，将其处理为真正的Java类）
  4. resolveClass（若加载成功，则链接指定的Java类）

URLClassLoader

URLClassLoader继承了ClassLoader，且是默认加载器AppClassLoader的父类，URLClassLoader提供了加载远程资源的能力，在写漏洞利用的payload或者webshell的时候我们可以使用这个特性来加载远程的jar来实现远程的类方法调用。

利用TemplatesImpl加载字节码

defineClass可以将字节码注册为类，但是由于defineClass方法是受保护的，只能在ClassLoader的子类中使用，而且需要足够的权限和访问级别。Class.forName方法并不提供直接访问ClassLoader的defineClass方法的途径
但是有一些Java底层的类用到了它，如TemplatesImpl

TemplatesImpl类是Java XSLT编译器(XML到另一个XML转换的规则和操作)的一个实现类，负责将XSLT模板转换成XML数据，定义了创建和操作XSLT模板的方法。
