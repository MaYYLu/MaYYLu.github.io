身份认证和数据安全

Author: mayylu Date: 二月 25, 2023  

前言

记录一下遇到的身份认证技术，不定时更新

随机数

php伪随机数

  1. mt_rand()函数
    使用语法：mt_rand()默认0到9; or mt_rand(min,max)生成一个区间内的随机数;需要mt_srand()函数来获取种子

漏洞:
当随机种子生成后，后面生成的随机数都会根据这个随机种子生成。所以同一个种子下，随机数的序列是相同的

所以只要获取种子就能知道特点情况下mt_rand()函数生成的值
使用 php_mt_seed工具获取种子

Token

Token产生过程：
当客户端请求服务端页面时，服务端会生成一个随机数Token存放在Session中，同时也会将session发送给客户端（一般通过构造hidden隐藏在表单中），下次客户端提交请求时，Token会随着表单一起提交到服务器端。

Token和session的区别

  1. 普通的token需要存放在服务器端的数据库，但是认证的token可以不存到数据库，如jwt
  2. Token是自定义设置的(可以防csrf)，可以放在任何位置，session固定位置

JWT

无状态的身份验证,我们的服务器不需要存储 Session 信息。这显然增加了系统的可用性和伸缩性，大大减轻了服务端的压力。

Json Web Token由以下三部分组成：

  • Header（头部） 该JSON以Base64Url加密（可以被对称解密）后形成了JWT的第一部分：eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9
  • Payload（载荷）
  • Signature（签名）
    Signature需要将base64加密后的header和payload使用.连接，然后通过header所使用的加密方式进行加盐（secret）组合加密，产生了jwt的第三部分。以使用HMAC SHA256算法为例，signature以如下方式产生： 算出签名以后，把 Header、Payload、Signature 三个部分拼成一个字符串，每个部分之间用”点”（.）分隔，就可以返回给用户。

Flask Session 伪造

Flask作为小型框架默认存储方式是使用 cookie，flask的这种session机制，可以节省服务器的开销，因为把所有的信息都存储到了客户端（浏览器）。但也可以通过配置将其保存在服务器端。
flask中的session机制是：把敏感数据经过加密后放入session中，然后再把session存放到cookie中，下次请求的时候，再从浏览器发送过来的cookie中读取session，然后再从session中读取敏感数据，并进行解密，获取最终的用户数据。

在 Flask 中，默认情况下使用 Werkzeug 提供的加密算法。Werkzeug 使用了一种称为“签名序列化”的技术将 Session 数据进行编码和解码，并使用一个密钥对 Session 进行签名以保证其完整性和安全性。具体地说，Werkzeug 使用 HMAC 哈希算法对 Session 进行签名以防止数据篡改，同时使用 AES 加密算法对 Session 进行加密以保护其机密性。
这就导致flask的session可看不能改
